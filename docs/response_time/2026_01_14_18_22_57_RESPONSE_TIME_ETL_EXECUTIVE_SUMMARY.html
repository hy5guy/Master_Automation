Problem

Duplicate officer responses inflated counts and skewed averages.

Primary Fix

  • Deduplicate by ReportNumberNew. Keep the first officer response per call.
  • Run deduplication before any response time calculations.

Code Changes

  • **New ETL Script:** `response_time_monthly_generator.py` (v2.0.0)
  • **Configuration File:** `config/response_time_filters.json`
  • **Processing Pipeline:** Expanded from 6 steps to 12 steps

Data Changes

  • **Deduplication:** Each call counted once (not per officer)
  • **Filtering:** Only relevant dispatched calls included
  • **Metrics:** More accurate representation of actual response times
  • **Coverage:** Full 13-month range (Dec 2024 - Dec 2025)

Processing Pipeline (12 Steps)

  • **Deduplication** by ReportNumberNew (removes officer duplicates)
  • **How Reported filter** (excludes Self-Initiated)
  • **YearMonth creation** from date fields
  • **Date range filter** (2024-12 to 2025-12)
  • **Admin incident filter** (existing administrative filters)
  • **Response time calculation** (Time Out - Time Dispatched)
  • **Time window filter** (0-10 minutes only)
  • **Response Type + Category_Type mapping** (from CallType_Categories.csv)
  • **Category_Type filter** (with inclusion overrides)
  • **Specific incident filter** (42 administrative incidents)
  • **Data verification** (quality checks)
  • **Final validation** (valid Response Type only)
