**Date**: February 9, 2026

**Status**: ✅ Ready for Power BI Implementation

---

## 🔥 Issue Summary

### Problems Encountered
Power BI query `___ResponseTimeCalculator` was failing with multiple errors:

**Error 1: DataSource.NotFound**
```
DataSource.NotFound: File or Folder: Could not find a part of the path 
'C:\Dev\PowerBI_Data\Backfill\2025_10\response_time\2025_10_Average Response Times  Values are in mmss.csv'
```

**Error 2: Duplicate YearMonth (v2.1.0)**
```
Expression.Error: The field 'YearMonth' already exists in the record.
```

**Error 3: Duplicate Response_Type (v2.1.1)**
```
Expression.Error: The field 'Response_Type' already exists in the record.
```

### Root Cause
1. M code was using **hardcoded file paths** from the old system that no longer exist
2. CSV files generated by ETL already contain columns with standardized names (`Response_Type`, `Response_Time_MMSS`)
3. CSV files already contain calculated columns (`YearMonth`, `Date_Sort_Key`, `Average_Response_Time`)
4. M code attempted to rename and add these columns again, causing duplicate field errors

---

## ✅ Solution: Dynamic File Loading with Intelligent Column Handling (v2.1.2)

### New Approach (v2.1.2)
The updated M code **automatically discovers and loads all available monthly response time CSV files** with **comprehensive column handling**:

1. **Scans** the Backfill directory recursively
2. **Finds** all CSV files in `response_time` subdirectories
3. **Checks** if columns already exist with correct names before renaming (prevents rename conflicts)
4. **Checks** if calculated columns exist before adding them (prevents duplicate columns)
5. **Loads** each file automatically
6. **Combines** all months into a single dataset
7. **Adapts** as new monthly files are added

**Benefits**:
- ✅ No more hardcoded paths
- ✅ No duplicate column errors (renaming or adding)
- ✅ Works with any number of months (13-month rolling window or more)
- ✅ Automatically picks up new monthly files
- ✅ Compatible with timereport hybrid source strategy
- ✅ Works with ETL-generated CSVs (regardless of which columns are pre-calculated)
- ✅ No manual M code updates needed when adding new months
- ✅ Handles both legacy CSVs and new ETL-generated CSVs seamlessly

2. Calculates response times for each month

1. Scans all `response_time` subdirectories in Backfill

- No M code updates needed when adding new months

## 🚨 Troubleshooting

### Error: "Expression.Error: The field 'Response_Type' already exists"
**Symptom**: M code fails at LoadedFiles step with duplicate field error for Response_Type

**Root Cause**: CSV files generated by Python ETL already use standardized column names (`Response_Type` instead of `"Response Type"`)

**Solution**: ✅ **Fixed in v2.1.2**
- M code now checks if `Response_Type` column exists before attempting to rename `"Response Type"` → `Response_Type`
- Same logic applied to `Response_Time_MMSS` and `MM-YY` columns
- Uses conditional renaming: only renames if target column doesn't already exist
- Works with both legacy CSVs (with spaces in column names) and new ETL CSVs (with underscores)

**Verification**:
Check your CSV column names:
```powershell
# Check CSV column headers
(Import-Csv "C:\Users\carucci_r\OneDrive - City of Hackensack\PowerBI_Data\Backfill\2026_01\response_time\2026_01_Average_Response_Times__Values_are_in_mmss.csv" |
  Get-Member -MemberType NoteProperty).Name
```

If you see `Response_Type` (with underscore) instead of `"Response Type"` (with space), you need v2.1.2 or later.

---

### Error: "Expression.Error: The field 'YearMonth' already exists"
**Symptom**: M code fails at LoadedFiles step with duplicate field error

**Root Cause**: CSV files generated by Python ETL already contain calculated columns (`YearMonth`, `Date_Sort_Key`, `Average_Response_Time`)

**Solution**: ✅ **Fixed in v2.1.1**
- M code now checks if columns exist before adding them
- Uses `Table.HasColumns()` to conditionally add columns
- Works with both old CSVs (without calculated columns) and new CSVs (with calculated columns)

**Verification**:
Check your CSV files to see if they already have these columns:
```powershell
# Check CSV structure
Import-Csv "C:\Users\carucci_r\OneDrive - City of Hackensack\PowerBI_Data\Backfill\2026_01\response_time\2026_01_Average_Response_Times__Values_are_in_mmss.csv" |
  Get-Member -MemberType NoteProperty |
  Select-Object Name
```

If you see `YearMonth`, `Date_Sort_Key`, or `Average_Response_Time` in the output, you need v2.1.1 or later.
