Overview

This document provides technical details for the monthly report data pipeline: script paths, input/output locations, shared folder structure, and Power BI integration. For a non-technical overview, see Monthly_Report_Data_Population_Executive_Summary.html.

ETL Scripts (Execution Order)

Order Script Name Script File Output Location
1 Arrests arrest_python_processor.py 01_DataSources\ARREST_DATA\Power_BI\
2 Community Engagement src\main_processor.py 02_ETL_Scripts\Community_Engagment\output\
3 Overtime TimeOff overtime_timeoff_13month_sworn_breakdown_v10.py 02_ETL_Scripts\Overtime_TimeOff\output\ and analytics_output\

ETL Input Sources

  • Arrests: 01_DataSources\ARREST_DATA\ — Lawsoft monthly export (.xlsx or .tab)
  • Community Engagement: Source workbooks in 02_ETL_Scripts\Community_Engagment\
  • Overtime TimeOff: 05_EXPORTS\_Overtime\, _Time_Off\, Backfill for prior months
  • Response Times: 05_EXPORTS\_CAD\ or timereport exports — CAD data per month
  • Summons: 05_EXPORTS\_Summons\E_Ticket\YYYY\YYYY_MM_eticket_export.csv

Shared Folder Structure (Contributions)

Base path: Shared Folder\Compstat\Contributions\

  • Chief: Chief\chief_monthly.xlsx
  • CSB: CSB\csb_monthly.xlsm
  • Detectives: Detectives\detectives_monthly.xlsx
  • Drone: Drone_Monthly.xlsx
  • ESU: ESU\ESU.xlsx
  • Patrol: Patrol\patrol_monthly.xlsm
  • REMU: REMU\REMU_Monthly.xlsx
  • STACP: STACP\STACP.xlsm
  • Traffic: Traffic\Traffic_Monthly.xlsx

Other Data Sources

  • NIBRS: 01_DataSources\NIBRS\NIBRS_Monthly_Report.xlsx
  • Benchmark (Use of Force, etc.): 05_EXPORTS\Benchmark\ — CSV exports
  • TAS/CAD (SSOCC): 05_EXPORTS\_CAD\tas\yearly\ — dispatcher incident data

Power BI Integration

  • Report template: 15_Templates\Monthly_Report_Template.pbix
  • Monthly report output: Shared Folder\Compstat\Monthly Reports\YYYY\MM_monthname\YYYY_MM_Monthly_Report.pbix
  • Power BI drop folder: PowerBI_Date\_DropExports\ — ETL outputs copied here automatically
  • Report month parameter: pReportMonth — set to first day of report month (e.g., 2/1/2026)

Orchestration

Command Purpose
.\scripts\run_all_etl.ps1 Run all enabled ETL scripts in order
.\scripts\run_all_etl.ps1 -DryRun Validate inputs without executing
.\scripts\run_etl_script.ps1 -ScriptName "Arrests" Run a single script by name

Configuration: Master_Automation\config\scripts.json
