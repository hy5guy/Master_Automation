- Null Category_Type values (exclude null values, log warning)

- Case sensitivity (use exact Category_Type match, normalize incident names)

**Code Pattern:**
```python
# Step 6.5: Filter by Category_Type (with inclusion overrides)
logger.info("=" * 70)
logger.info("STEP 6.5: Filter Category_Type (with inclusion overrides)")

excluded_categories = set(filter_config['filters']['category_types']['exclude'])
inclusion_overrides = filter_config['filters']['inclusion_overrides']['include_despite_category_filter']
inclusion_overrides_normalized = {normalize_incident_name(i) for i in inclusion_overrides}

# Mark records to keep (either not excluded OR in override list)
df['_keep_override'] = df['Incident_Normalized'].isin(inclusion_overrides_normalized)
before_category = len(df)

# Filter: keep if in override list OR category is not excluded
df = df[
    df['_keep_override'] | ~df['Category_Type'].isin(excluded_categories)
].copy()

# Drop temporary column
df = df.drop(columns=['_keep_override'], errors='ignore')

logger.info(f"Category_Type filter (exclude {len(excluded_categories)} categories, {len(inclusion_overrides)} overrides): {before_category:,} -> {len(df):,} records")
```

## Edge Cases to Handle

### Data Quality Edge Cases
1. **Missing Columns:**
   - "How Reported" column missing (log warning, skip filter)
   - "Category_Type" missing from mapping (raise error)
   - "ReportNumberNew" missing (log warning, skip deduplication)

2. **Null/Empty Values:**
   - Null "How Reported" values (keep, only exclude exact matches)
   - Null Category_Type after merge (log warning, exclude)
   - Null Response_Type after merge (log warning, exclude later)
   - Null incident names (keep, normalize to "UNKNOWN")

3. **Data Type Issues:**
   - Non-string "How Reported" values (convert to string)
   - Non-string incident names (convert to string)
   - Invalid date formats (log error, set to null)

4. **Filtering Edge Cases:**
   - All records filtered out (log error, return empty dataframe)
   - Empty exclusion lists (keep all records)
   - Empty inclusion override list (no overrides)
   - Case sensitivity mismatches (use normalized names)

5. **Mapping Edge Cases:**
   - Duplicate Incident_Key values in mapping (keep first, log warning)
   - Unmapped incidents (log warning, will be filtered out)
   - Missing Category_Type in mapping (raise error)

6. **Config File Edge Cases:**
   - Config file not found (raise error with helpful message)
   - Invalid JSON (raise error with JSON parse error)
   - Missing required keys (raise error listing missing keys)
   - Empty exclusion lists (handle gracefully, log info)

### Performance Edge Cases
1. **Large Datasets:**
   - Memory usage (use `.copy()` judiciously, avoid unnecessary copies)
   - Processing time (log progress for large datasets)

2. **Empty Results:**
   - No records after filtering (log warning, return empty dataframe)
