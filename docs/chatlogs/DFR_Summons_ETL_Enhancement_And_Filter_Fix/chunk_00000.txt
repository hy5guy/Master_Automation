## Changes

### 1. **FilteredStatus step**
The `FilteredStatus` step was missing from the saved M code. It’s now added between `FilteredRecalls` and `FilteredData`. ### 2. **More robust status filtering**
The old logic used exact matches (`cleaned <> "dismissed"` and `cleaned <> "void"`), so values like **"Voided"** were not excluded. The new logic uses `Text.Contains` so any status containing "dismiss" or "void" is excluded:

- **"Dismissed"** → excluded (contains "dismiss")
- **"Void"** → excluded (contains "void")
- **"Voided"** → excluded (contains "void")
- **"Request to Dismiss"** (if it ever appears in status) → excluded

### 3. **Null handling**
`status ? ? ""` is used so null status values are handled safely before `Text.Lower` and `Text.Trim`. ### 4. **Filter chain**
`FilteredData` now filters from `FilteredStatus` instead of `FilteredRecalls`, so both recall and status filters are applied. ---
