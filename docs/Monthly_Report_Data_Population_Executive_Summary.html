Monthly Report Data Population — Executive Summary

Internal Briefing  |  March 2026  |  Prepared for Monthly Report Stakeholders
Prepared by: R. A. Carucci, Principal Analyst Date: March 9, 2026 Subject: How Data Flows Into the Monthly CompStat Report Status: Current as of March 2026

Executive Summary

The Hackensack PD Monthly CompStat report is built from two types of data sources: automated scripts that process raw exports and produce ready-to-use files, and shared folders where bureaus and units maintain Excel workbooks with their monthly activity. Power BI pulls from both to assemble the final report.

Bottom line: Roughly half of the report’s data comes from automated scripts; the rest comes from shared Excel workbooks that responsible units update each month.

Data Populated by Automated Scripts

A central automation system runs several scripts each month. These scripts take raw exports (from Lawsoft, CAD, e-ticket, time systems, etc.), clean and aggregate the data, and write CSV or Excel files into a designated drop folder. Power BI then reads these files when the report is refreshed.

Report Area What the Script Does
Arrests Processes Lawsoft arrest exports and produces arrest categories, distributions, and top-five summaries.
Community Engagement Combines outreach and engagement data from multiple sources into a single dataset for bureau-level visuals.
Overtime & Time Off Processes time-report exports to compute accrual, usage, and monthly breakdowns for sworn and non-sworn personnel.
Response Times Uses CAD/timereport data to calculate average response times by priority (Emergency, Urgent, Routine) with proper filtering and deduplication.
Summons Processes e-ticket exports to produce department-wide and bureau-level summons counts, including moving and parking violations.

Scripts run in a defined order each month. Outputs are automatically copied to the Power BI drop folder. No manual file copying is required for these areas.

Data Populated by Shared Folders

All other report areas are populated by Excel workbooks stored in shared folders. Designated personnel in each bureau or unit update these workbooks with their monthly activity. Power BI connects directly to these files and reads the data when the report is refreshed.

Report Area Responsible Unit / Source
Chief’s Projects & Initiatives Chief’s office — monthly workbook in shared folder
Crime Suppression Bureau (CSB) CSB — monthly activity workbook
Detective Division Detectives — case tracking and clearance workbook
Drone Program Drone unit — monthly activity workbook
ESU (Emergency Services Unit) ESU — monthly activity and tracked items workbook
Patrol Division Patrol — monthly activity workbook
REMU REMU — monthly workbook
STACP (Social Media, Training, etc.) STACP — combined workbook (social media, training, other activity)
Traffic Bureau Traffic — monthly activity workbook
NIBRS Data sources folder — NIBRS monthly report export
Benchmark (Use of Force, etc.) Exports folder — benchmark CSV exports
Policy & Training (Cost of Training) Policy Training ETL reads from source workbook; outputs feed Power BI

Each bureau or unit is responsible for updating its workbook by the monthly deadline. The shared folder structure ensures that Power BI always reads the latest version when the report is refreshed.

Monthly Workflow Overview

Each month, the process follows a consistent sequence: (1) raw exports are placed in their designated locations; (2) automated scripts run and produce outputs; (3) bureaus update their shared-folder workbooks; (4) the report month parameter is set in Power BI; (5) all queries are refreshed; and (6) the monthly report is saved and published.

Key takeaway: Successful monthly reports depend on timely raw exports, script execution, and bureau updates to shared workbooks. All three must be completed for the report to reflect the full month’s data.

Where to Get Help

For questions about script execution, data sources, or shared-folder locations, contact the Principal Analyst. For questions about what data to enter in a specific bureau workbook, contact the responsible unit lead.
